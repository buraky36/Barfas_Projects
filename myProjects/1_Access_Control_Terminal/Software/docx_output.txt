 Project Name: Multifunction Standalone Access Control Introduction : The device is a single door multifunction standalone Access controller or a Wiegand output reader. It uses Xtensa® dual-core 32-bit LX6 microprocessor assuring high and stabe performance. The operation is very user-friendly. Features: PIN Lenght: 4~6 digits Communacation: Wiegand Input*1, Wiegand-Output*1 Control Output: Electric Lock*1 Control Input: Door Sensor*1, Exit Button*1 Operation Modes : The device has 3 different operating modes. Standalone Mode (Default Mode) : The device can work as Standalone Access Control for single door. Bu modda Wiegand Input/Output haberleşmesi olmayacak. Digital Inputlar bu modda çalışacak. 1.Dig_In : Exit Button : Direkt olarak kapıyı açabilir. 2.Dig_In : Door Contact: Kapının Açık/Kapalı konumu algılamak için kullanacağız. Bu giriş LOW olduğunda kapı açık konumda olduğunu anlayacağız. Controller Mode: The device can work as Controller, connected with the external Wiegand Reader. Bu modun Standalone modundan tek farkı Wiegand girişinden harici bir reader bağlanabilecek. Bu modda cihaza Wiegand girişinden harici bir Reader bağlanabilecek. Yani dahili inputları olan RFID, PIN ve QR okuyucularının yanı sıra, harici bağlanan Reader da UID gönderebilecek. Bu mod set edildiğinde Wiegand Output Enable- Low a çekilmeli ki Wiegand direction-input olsun. Cihaz bu moda set edildiği zaman "Menu 9" Wiegand Data Formats ayarlayabilecek. Bu data formats alınacak Wiegand paketlerinin formatı olacak. Digital Inputlar bu modda çalışacak. 1.Dig_In : Exit Button : Direkt olarak kapıyı açabilir. 2.Dig_In : Door Contact: Kapının Açık/Kapalı konumu algılamak için kullanacağız. Bu giriş LOW olduğunda kapı açık konumda olduğunu anlayacağız. Wiegand Reader Mode : The device can work as Standard Wiegand Reader, connected to the third party Controller. Cihaz bu moda set edildiği zaman sadece "Menu 9" konfigürasyonu yapılabilecek. Yani Wiegand Data Formats ayarlanabilecek. Bu data formats gönderilecek Wiegand paketlerinin formatı olacak. Çünkü cihaz bu modda inputlarından okuduğu ID leri hiç bir işlem yapmadan Wiegand Outputundan gönderecek. Bunu yapmadan önce Wiegand Output Enable pinini High yapacak. Bu pin bu modda sürekli High olacak. Bunu Wiegand direction-Output olarak ayarlamak için yapacak. Digital Inputlar bu modda çalışmayacak. Bluetooth BLE Anahtar Kavramlar •deviceId: deviceId parametresi cihazın benzersiz kimliğini temsil eder. •pairKey / deviceKey: BLE eşleşmesi sırasında üretilen çift ve cihaz anahtarlarıdır; sonraki HTTP isteklerinde kullanılır. •macAddress: Cihazın MAC Adresidir. •rssi: Kablosuz ağın sinyal gücünü belirtir (Received Signal Strength Indicator). •frequency: Kablosuz ağın frekans bandını belirtir (2.4 GHz, 5 GHz). •name: Kablosuz ağın adıdır (SSID). Bluetooth Name ve DeviceID için aşağıdaki konfigürasyonu kullanalım: String ble_name; void initDeviceId() { // WiFi.macAddress() bazen 0 dönebilir, bu yüzden EfuseMac kullanıyoruz. uint64_t chipid = ESP.getEfuseMac(); char idStr[32]; // chipid'nin alt 3 byte'ını (24 bit) alarak unique ID oluştur. // Genellikle MAC adresinin son kısmına denk gelir. uint32_t low = (uint32_t)chipid; sprintf(idStr, "SMART_%06X", (low &amp; 0xFFFFFF)); // SMART_XXXXXX formatı device_id = String(idStr); char nameStr[64]; sprintf(nameStr, "Onloi_%s", idStr); ble_name = String(nameStr); DEBUG_PRINT("Generated Device ID: "); DEBUG_PRINTLN(device_id); DEBUG_PRINT("Generated BLE Name: "); DEBUG_PRINTLN(ble_name); } BLE Komut Değerleri handshake : Bağlantı kurulumu sırasında gerçekleştirilen kimlik doğrulama sürecidir. pair : Uygulama ile cihaz arasında güvenli bağlantı kurmak için kullanılır. getSsids : Kullanılabilir Wi-Fi ağ listesini (SSID) çeker. connectWifi : Wi-Fi bağlantısının kurulup kurulmadığını kontrol eder.
