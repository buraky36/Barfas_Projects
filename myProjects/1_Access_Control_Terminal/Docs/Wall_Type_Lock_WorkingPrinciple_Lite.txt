Online Operation:
Bölüm1:
BLE (BLUETOOTH LOW ENERGY) PROTOKOLÜ : Bluetooth privisioning işlemini daha önce yazılıma eklemiş olabiliriz. Kontrol et.
Bu bölüm, cihazın mobil uygulama ile Bluetooth Low Energy (BLE) üzerinden gerçekleştirdiği eşleşme ve Wi-Fi yapılandırma protokolünü tanımlar.
1.1: BLE Anahtar Kavramlar:
•deviceId: deviceId parametresi cihazın benzersiz kimliğini temsil eder.
•pairKey / deviceKey: BLE eşleşmesi sırasında üretilen çift ve cihaz anahtarlarıdır; sonraki HTTP isteklerinde kullanılır.
•macAddress: Cihazın MAC Adresidir.
•rssi: Kablosuz ağın sinyal gücünü belirtir (Received Signal Strength Indicator).
•frequency: Kablosuz ağın frekans bandını belirtir (2.4 GHz, 5 GHz).
•name: Kablosuz ağın adıdır (SSID).
Bluetooth Name ve DeviceID için aşağıdaki konfigürasyonu kullanalım:

String ble_name;

void initDeviceId() {
  // WiFi.macAddress() bazen 0 dönebilir, bu yüzden EfuseMac kullanıyoruz.
  uint64_t chipid = ESP.getEfuseMac();

  char idStr[32];
  // chipid'nin alt 3 byte'ını (24 bit) alarak unique ID oluştur.
  // Genellikle MAC adresinin son kısmına denk gelir.
  uint32_t low = (uint32_t)chipid;

  sprintf(idStr, "SMART_%06X", (low & 0xFFFFFF)); // SMART_XXXXXX formatı
  device_id = String(idStr);

  char nameStr[64];
  sprintf(nameStr, "Onloi_%s", idStr);
  ble_name = String(nameStr);
  DEBUG_PRINT("Generated Device ID: ");
  DEBUG_PRINTLN(device_id);
  DEBUG_PRINT("Generated BLE Name: ");
  DEBUG_PRINTLN(ble_name);
}

handshake : Bağlantı kurulumu sırasında gerçekleştirilen kimlik doğrulama sürecidir.
pair : Uygulama ile cihaz arasında güvenli bağlantı kurmak için kullanılır.
getSsids : Kullanılabilir Wi-Fi ağ listesini (SSID) çeker.
connectWifi : Wi-Fi bağlantısının kurulup kurulmadığını kontrol eder.

1. Cihaz en başından ve ya daha sonradan(yani offline çalıştı ve kullanıcılar lokal database e kaydedilmişti.) online moda geçebilir.

3. Online modda çalıştıktan sonra tekrar offline modda çalışmak istenirse cihaz resetlenecektir.

RGB Led - WS2812B ve Buzzer Senaryoları:

	1. Working Mode

- After the access controller is powered on, its LED status indicator turns green and blinks for 1 time firstly. Then it turns red and blinks
for 3 times, and the buzzer buzzes a beep sound indicating the initialization process is completed.

- When using the access control terminal, it will send out different sound prompt and the LED indicator on it has different statues. For the
detailed information, refer to the table below.

Sound Prompt		Led Indicator						Description
					Green and Blinking Slowly			The access controller is working normally.
One Beep												Pressing keys prompt, Swiping card prompt, Swiping QR prompt
Two Rapid Beeps		Solid Green							The operation of pressing keys, swiping card or swiping QR valid.
Three Slow Beeps	Solid Red							The operation of pressing keys, swiping card or swiping QR invalid.
Two Rapid Beeps											The operation of entering the programming mode is succesful.
Three Slow Beeps										The operation of entering the programming mode is failed.
Beep for 10 Seconds										Delayed door alarm (the door is opened over the time limitation)

	2. Programming  Mode
- After entering the programming mode, LED status indicator turns blue and blinks rapidly.
- After exiting the programming mode(with "*#") , the buzzer buzes two rapid beeps for switching the programming mode to the normalde mode.

	2.1. Update Master Code
	
Sound Prompt		Led Indicator						Description
3 Slow Beeps		Solid red during the beeps			The operation of update master code is failed.
2 Rapid Beeps		Solid green during the beeps		The operation of update master code is successful.

	2.2. Add Master Card
	
Sound Prompt		Led Indicator						Description	
3 Slow Beeps									Enter another user ID for the current user ID has been applied.
2 Rapid Beeps									The user ID is valid, wait for swiping.
1 Beep											Swipe the card or QR.

	2.3. Adding User

2 Rapid Beeps		Solid green during the beeps		The user ID is valid, wait for swiping.
