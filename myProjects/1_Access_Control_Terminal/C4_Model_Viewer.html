C4 Model: Smart Access Control Terminal

This document outlines the architecture of the Smart Access Control Terminal using the C4 model framework. You can use your browser's Print feature (Ctrl+P or Cmd+P) to "Save as PDF".

Level 1: System Context Diagram

The System Context diagram provides a high-level view of the Access Control Terminal and how it interacts with its environment, including users and external systems.

C4Context title System Context for Access Control Terminal Person(personnel, "Personnel", "Employee or visitor requesting access") Person(admin, "Administrator", "Manages device configuration via Mobile App") System(act_system, "Access Control Terminal", "Smart device that authenticates users and controls physical access") System_Ext(central_server, "Central Server", "Remote backend server for log storage, central auth, and configs") System_Ext(door_hardware, "External Door Hardware", "Physical door locks, exit buttons, and door sensors") System_Ext(master_controller, "Master Controller (Optional)", "External Wiegand/OSDP controller when device is in Reader Mode") Rel(personnel, act_system, "Presents RFID, QR code, or PIN to") Rel(admin, act_system, "Provisions Wi-Fi and configures via BLE using Mobile App") Rel(act_system, central_server, "Sends event logs and fetches config/auth via HTTPS", "Wi-Fi (HTTP/S)") Rel(act_system, door_hardware, "Controls locks, monitors sensors via GPIO") Rel(act_system, master_controller, "Sends badge data to", "Wiegand / OSDP")

Level 2: Container Diagram

The Container diagram zooms into the Access Control Terminal to show the high-level executable/storage containers running on the ESP32 microcontroller.

C4Container title Container Diagram for Access Control Terminal System_Boundary(act_system, "Access Control Terminal") { Container(firmware, "Firmware Application", "C / ESP-IDF (FreeRTOS)", "Handles hardware interactions, business logic, auth, and state management") ContainerDb(nvs_storage, "NVS Storage", "Flash Memory", "Stores device configuration (Network, Keys, Mode) and offline logs") } System_Ext(central_server, "Central Server", "Remote backend server") System_Ext(door_hardware, "Door Hardware", "Locks & Sensors") Person(personnel, "Personnel", "User") Person(admin, "Administrator", "Admin Configurator") Rel(personnel, firmware, "Interacts by swiping card, typing PIN, scanning QR") Rel(admin, firmware, "Configures via BLE") Rel(firmware, nvs_storage, "Reads/Writes Configs & Logs", "NVS API") Rel(firmware, central_server, "Sends/Receives Data", "JSON/HTTPS") Rel(firmware, door_hardware, "Sends electrical signals", "GPIO")
